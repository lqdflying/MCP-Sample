# GitHub OAuth Callback Guide

This guide explains which callback URL to use when connecting this MCP server to GitHub OAuth, especially for Power Automate custom connectors.

## Choose the Correct Callback URL

The callback URL depends on which OAuth flow is driving the sign-in.

### 1. Power Automate Custom Connector OAuth (recommended for this project)

Use the redirect URL generated by Power Automate after you save or update the custom connector.

- In Power Automate custom connector Security tab, choose OAuth 2.0 and GitHub.
- Save or update connector.
- Copy the generated Redirect URL.
- In your GitHub OAuth App settings, set Authorization callback URL to that exact Power Automate Redirect URL.

For this flow, GitHub redirects back to Power Automate, then Power Automate sends bearer tokens to this MCP server.

### 2. FastMCP built-in OAuth proxy flow (direct server-driven OAuth)

Use this server callback path:

- https://your-domain/auth/callback

Example:

- https://githubmcp.lqdflying.info/auth/callback

## Important Limitation

GitHub OAuth Apps typically use a single callback URL per app configuration.

If you need both flows at the same time:

1. Create one GitHub OAuth App for Power Automate callback URL.
2. Create another GitHub OAuth App for server callback URL (/auth/callback).

## Recommended Setup For This Repository

If your main goal is Power Automate:

1. Use the Power Automate Redirect URL as the GitHub OAuth callback URL.
2. Keep this MCP server in `MCP_AUTH_MODE=both` if you also need token-based clients.
3. Set `ALLOWED_GITHUB_LOGINS` to your username for personal-only OAuth access.

## Quick Config Checklist

1. In `.env`, set:
2. `MCP_AUTH_MODE=both` (or `oauth` if token auth is not needed)
3. `GITHUB_CLIENT_ID=<your app client id>`
4. `GITHUB_CLIENT_SECRET=<your app secret>`
5. `BASE_URL=https://your-public-mcp-domain`
6. `ALLOWED_GITHUB_LOGINS=<your-github-login>`
7. In GitHub OAuth app, set callback URL based on chosen flow above.
